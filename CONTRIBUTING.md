# 🌱 Contribution Workflow

We follow the GitHub Flow workflow for all contributions.

## 📌 Steps to Contribute

### 1️⃣ Fork the Repository

Click the **Fork** button on GitHub to create your own copy of the repository.

### 2️⃣ Clone Your Fork

```bash
git clone https://github.com/your-username/WebDev-Cohort-2026.git
````

---
### 3️⃣ Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

Example:

```bash
git checkout -b feat/add-react-notes
```

---

### 4️⃣ Make Your Changes

Add your improvements, fixes, notes, or projects.

---

### 5️⃣ Commit Your Changes

We use **Conventional Commits** for commit messages.

```bash
git commit -m "feat: add authentication notes"
```

---

### 6️⃣ Push Your Branch

```bash
git push origin feature/your-feature-name
```

---

### 7️⃣ Open a Pull Request

Create a Pull Request from your branch to the `main` branch.

---

# 📌 Contribution Guidelines

Please follow these guidelines while contributing:

* Create separate branches for separate changes
* Keep pull requests clean and focused
* Write meaningful commit messages
* Avoid unrelated changes in one PR
* Update documentation when necessary
* Test your code before submitting

These practices help keep the repository organized and easier to maintain.

---

# 📝 Conventional Commit Examples

```bash
feat: add react hooks examples
fix: correct typo in backend notes
docs: update contribution guide
refactor: improve folder structure
style: format markdown files
```

📚 Learn more about Conventional Commits:

[https://www.conventionalcommits.org/](https://www.conventionalcommits.org/)
