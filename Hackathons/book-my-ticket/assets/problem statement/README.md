# 🎯 Hackathon Assignment: Book My Ticket (Backend)

This hackathon is a follow-up assignment based on your recent **Chai Aur SQL** class learnings.

---

## 📦 Starter Code

You will be provided with a starter source code that already contains working endpoints.

🔗 Starter Repository:
https://github.com/chaicodehq/book-my-ticket

---

## 🧠 Objective

Your task is **NOT** to build the system from scratch.

Instead, you must:

* Carefully study the existing codebase
* Extend it by adding authentication
* Implement a protected booking system

This simulates a real-world scenario where developers work on **existing production codebases**.

---

## 🚀 Project Goal

Build a simplified **Book My Ticket platform** where:

* 👤 Users can register
* 🔐 Users can login
* 🛡️ Only authenticated users can access protected endpoints
* 🎟️ Logged-in users can book seats for a movie

---

## 🎬 Scope

* Movie data can be **mocked**
* Focus mainly on:

  * Authentication system
  * Seat booking logic

---

## 💻 Frontend

* Frontend is **optional**
* Evaluation will be based on **backend implementation**

---

## 📏 Rules & Guidelines

* ✅ Starter source code must be used as the base
* ❌ Do NOT remove or break existing endpoints
* ➕ Add authentication layer on top of the current system
* 🔑 Implement **register** and **login** functionality
* 🔒 Protect booking-related endpoints using auth middleware
* 👤 Only logged-in users should be allowed to book seats
* 🚫 Prevent duplicate seat bookings
* 🔗 Associate bookings with logged-in users
* 🎬 Keep movie data mocked for now
* 🧹 Code should be clean, structured, and readable

---

## 📤 Submission Guidelines

* Submit a **working GitHub repository**
* Include a proper **README.md** explaining:

  * Setup steps
  * Project flow
  * API usage

---

## 🏁 Goal of Assignment

This assignment is designed to help you:

* Work with existing codebases
* Understand backend architecture
* Implement real-world authentication
* Handle concurrency (seat booking)
* Write clean, production-like code
